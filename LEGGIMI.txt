========================================
 HOME APP — Istruzioni
 Versione 2.8.0
========================================

Home App è una web app statica (PWA leggera) per gestire lavori di casa,
stanze, piani/aree e idee progettuali. Funziona sia aprendo index.html in
locale sia pubblicata su GitHub Pages. I dati restano salvati SOLO sul tuo
dispositivo (localStorage del browser), nella chiave "casa-app-v2-data".

----------------------------------------
 FUNZIONALITÀ PRINCIPALI
----------------------------------------

- HOME: riepilogo (lavori, aperti, alta priorità, completati, idee) e le
  stanze raggruppate per piano/area.
- LAVORI: elenco con priorità (alta/media/bassa) e stato
  (da fare / in corso / in attesa / fatto), filtro per priorità.
  Scadenza facoltativa con badge "Scaduto / Oggi / Tra N gg"; in Home
  il numero di lavori "Scaduti".
  ORDINAMENTO: il pulsante "Ordina" (in alto a destra) apre un menu per
  scegliere come disporre i lavori: Manuale, Scadenza, Priorità, Stanza,
  Aggiunti di recente. La scelta viene ricordata. Con "Manuale" compare
  una maniglia su ogni card: trascinala per metterli nell'ordine che vuoi.
  COMPLETATI: i lavori segnati "fatto" escono dalla lista e finiscono in
  una sezione "Completati (N)" comprimibile in fondo, così la lista resta
  pulita.
  STATO RAPIDO: ogni nuovo lavoro parte come "Da fare". Lo stato
  (Da fare / In corso / In attesa / Fatto) si cambia con un tocco
  direttamente dalla pagina del lavoro, senza aprire la modifica.
- IDEE: elenco con stato (bozza / da valutare / approvata / archiviata),
  link di riferimento e stima costo. Stesso pulsante "Ordina" (Manuale /
  Stato / Costo / Recenti) con trascinamento in modalità Manuale; le idee
  "archiviata" finiscono in una sezione "Archiviate (N)" comprimibile.
- Dettaglio LAVORO/IDEA: titolo completo, CHECKLIST riordinabile
  (trascina la maniglia), e SPESA collegata.
- Un lavoro/idea può stare su PIÙ STANZE (oppure su nessuna: "Senza stanza").
- SPESA: lista della spesa unica, con quantità, categoria "Dove comprare"
  (ferramenta, arredamento, ...), nota per articolo e collegamento opzionale
  a una stanza, a un lavoro o a un'idea. Filtro per categoria. Tocca un
  articolo per modificarlo.
- OPZIONI: organizzate a menu (tocca una voce per entrare nella sezione):
    · Aspetto (tema chiaro/scuro, colore accento)
    · Gestione casa (piani/aree e stanze)
    · Backup e dati (esporta/importa JSON, backup Google Drive)
    · Diagnostica (stato dati e reset)
  La freccia in alto a sinistra riporta al menu Opzioni.

NOTA: la gestione delle STANZE è dentro la gestione dei PIANI/AREE
(Opzioni -> Gestione piani/aree -> tocca un piano -> aggiungi/gestisci
le sue stanze).

----------------------------------------
 1. COME CARICARE I FILE SU GITHUB PAGES
----------------------------------------

1. Crea una nuova repository su GitHub (o aprine una esistente).
2. Carica TUTTI i file di questo progetto nella ROOT della repository,
   cioè al primo livello, NON dentro una sottocartella.

   File da caricare (tutti allo stesso livello):
     - index.html
     - app.js
     - sw.js
     - manifest.json
     - base.css
     - layout.css
     - components.css
     - home.css
     - lavori.css
     - idee.css
     - spesa.css
     - stanza.css
     - opzioni.css
     - themes.css
     - icon-192.png
     - icon-512.png
     - LEGGIMI.txt

3. Vai su:  Settings  ->  Pages
4. In "Build and deployment", alla voce "Source" scegli "Deploy from a branch".
5. Seleziona il branch (di solito "main") e la cartella "/ (root)".
6. Salva. Dopo qualche minuto GitHub ti darà un link tipo:
       https://TUONOME.github.io/NOMEREPO/

   Apri quel link dall'iPhone per usare l'app.

IMPORTANTE:
- Tutti i file vanno nella ROOT. NON usare sottocartelle.
- NON caricare il file ZIP direttamente: GitHub Pages non lo apre.
  Devi caricare i singoli file (oppure estrarre lo ZIP e caricarne il
  contenuto).

----------------------------------------
 2. CARICARE I FILE DALL'IPHONE
----------------------------------------

Dall'app GitHub o dal sito github.com nel browser:
  - Apri la repository.
  - Usa "Add file" -> "Upload files".
  - Seleziona i file e conferma il commit.
Se carichi uno ZIP, estrailo prima (es. con l'app File di iPhone) e poi
carica i file estratti, NON lo ZIP.

----------------------------------------
 3. AGGIORNAMENTI E OFFLINE (automatici)
----------------------------------------

Da questa versione l'app usa un "service worker" che le permette di:
  - APRIRSI ANCHE SENZA RETE (offline): i file restano salvati sul
    dispositivo, così l'app parte anche in metropolitana o senza dati.
  - AGGIORNARSI DA SOLA: quando sei online prende sempre l'ultima
    versione pubblicata. Non serve più il vecchio trucco "?v=2".

Quando esce una nuova versione, in basso compare l'avviso
"Nuova versione disponibile" con il pulsante "Aggiorna": premilo e
l'app si ricarica già aggiornata. I tuoi dati NON vengono toccati.

PER CHI PUBBLICA: a ogni rilascio cambia la riga
   var CACHE_VERSION = "casa-app-2.8.0";
in cima al file sw.js (es. "casa-app-2.7.1"). È questo cambio che fa
scattare l'avviso di aggiornamento sui dispositivi. Ricordati di
caricare ANCHE sw.js insieme agli altri file su GitHub Pages.

Se proprio vuoi forzare un ricaricamento "manuale" puoi ancora
aggiungere ?v=2 (poi ?v=3, ?v=4...) in fondo all'indirizzo, ma di
norma non serve più.

----------------------------------------
 4. SAFARI IPHONE MOSTRA UNA VECCHIA VERSIONE
----------------------------------------

Di norma basta premere "Aggiorna" quando compare l'avviso di nuova
versione (vedi punto 3). Se ancora vedi una versione vecchia, prova:
  1. Tira la pagina verso il basso per ricaricare.
  2. Chiudi del tutto la scheda e riaprila.
  3. Apri l'indirizzo con ?v=2 (vedi punto 3), cambiando il numero.
  4. Impostazioni iPhone -> Safari -> "Cancella dati siti web e cronologia"
     (ATTENZIONE: questo può cancellare i dati salvati di altri siti).

NOTA: questa versione USA un service worker per funzionare offline e per
aggiornarsi da sola (vedi punto 3). La strategia è "prima la rete": online
prendi sempre l'ultima versione, offline parti dalla copia salvata.

----------------------------------------
 5. ESPORTARE E IMPORTARE I DATI (JSON)
----------------------------------------

I dati sono salvati solo nel browser del dispositivo. Per non perderli
(cambio telefono, pulizia dati, backup) usa Esporta/Importa.

ESPORTARE:
  - Apri l'app -> OPZIONI -> "Esporta JSON".
  - Viene scaricato un file .json: conservalo in un posto sicuro
    (es. app File, email a te stesso, cloud).

IMPORTARE:
  - Apri l'app -> OPZIONI -> "Importa JSON".
  - Seleziona un file .json esportato in precedenza.
  - Se il file è valido vedrai il messaggio "JSON importato".
  - Se il file non è valido vedrai "Errore JSON non valido" e i dati
    attuali NON verranno toccati.

Suggerimento: esporta un backup prima di fare un Reset dati.

----------------------------------------
 6. BACKUP AUTOMATICO SU GOOGLE DRIVE
----------------------------------------

Oltre all'export/import manuale, l'app può salvare i dati da sola su
Google Drive a ogni modifica, così hai sempre un backup nel cloud e puoi
recuperarlo su un altro dispositivo. Funziona anche dall'app installata
sulla Home di iPhone/iPad. Tutto avviene nel browser: niente server,
nessuna password salvata. Il backup è un file "casa-app-backup.json"
creato dall'app nel Drive di chi fa login (l'app vede SOLO quel file,
non il resto del tuo Drive).

IMPORTANTE - distribuzione ad amici/parenti:
Il "Client ID" qui sotto NON è una password e NON è per-utente: identifica
l'APP. Lo metti TU una volta sola dentro app.js. Chiunque usi l'app (tu,
amici, parenti) preme semplicemente "Collega Google Drive" e fa login col
PROPRIO account: il backup finisce sul SUO Drive. Nessuno deve inserire il
Client ID né configurare niente. Puoi quindi dare l'app già pronta.

NOTA: iCloud non è supportato perché Apple non offre un modo per scrivere
su iCloud Drive da una web app. OneDrive sarebbe possibile ma non incluso.

--- A. CONFIGURAZIONE (la fa SOLO chi pubblica l'app, una volta, gratis) ---

Serve un "Client ID" di Google. Lo crei col tuo account così:

  1. Vai su  https://console.cloud.google.com  e crea un progetto
     (es. "Home App").
  2. API e servizi -> Libreria -> cerca "Google Drive API" -> Abilita.
  3. API e servizi -> Schermata consenso OAuth:
       - Tipo utente: Esterno.
       - Compila nome app, email di assistenza e email sviluppatore.
       - Stato di pubblicazione: premi "PUBBLICA APP" -> stato
         "In produzione". Così CHIUNQUE può fare login, non solo gli
         utenti di test.
     IMPORTANTE: l'app usa lo scope "drive.file", che Google considera
     NON sensibile. Significa che, anche "in produzione", NON serve la
     verifica di Google: la pubblichi e funziona subito per tutti.
     (Se la lasci in stato "Test" funziona solo per le email che aggiungi
      a mano in "Utenti di test", max 100.)
  4. API e servizi -> Credenziali -> Crea credenziali ->
     ID client OAuth -> tipo "Applicazione web".
  5. In "URI di reindirizzamento autorizzati" incolla ESATTAMENTE
     l'indirizzo che l'app ti mostra:
       apri l'app pubblicata -> OPZIONI -> sezione
       "Backup cloud - Google Drive" -> sotto "Collega Google Drive"
       trovi la voce "Redirect URI da registrare".
     Sarà tipo:  https://TUONOME.github.io/NOMEREPO/index.html
     Copialo identico (deve combaciare carattere per carattere).
     NOTA: se distribuisci l'app da indirizzi diversi (es. ogni amico ha
     la sua repo GitHub), aggiungi qui TUTTI quegli indirizzi. In
     alternativa pubblica un'unica copia dell'app che usano tutti.
  6. Crea: copia l'ID client (finisce con .apps.googleusercontent.com).

--- B. INSERIRE L'ID NELL'APP (lo fa chi pubblica, una volta sola) ---

  1. Apri il file  app.js.
  2. Cerca la riga:   CLIENT_ID: ""
     (dentro CONFIG.GDRIVE, in alto nel file).
  3. Incolla il tuo ID tra le virgolette, es.:
       CLIENT_ID: "12345-abc.apps.googleusercontent.com"
  4. Salva e ricarica i file su GitHub (vedi punto 2).
  Da questo momento il pulsante "Collega Google Drive" funziona per
  chiunque apra quella copia dell'app, ognuno col proprio account.

--- C. USO QUOTIDIANO ---

  - OPZIONI -> "Collega Google Drive" -> accedi col tuo account.
  - Attiva "Backup automatico": da quel momento ogni modifica viene
    salvata su Drive (con un piccolo ritardo per non sovraccaricare).
  - "Esegui backup ora": forza subito un salvataggio.
  - "Ripristina da Drive": SOVRASCRIVE i dati locali con quelli del
    backup cloud (utile su un nuovo dispositivo). Chiede conferma.
  - "Scollega Drive": rimuove l'accesso da questo dispositivo.

--- D. COSE DA SAPERE ---

  - L'accesso dura circa 1 ora: ogni tanto l'app chiederà di ricollegarti
    con un tocco. È normale (non si possono tenere sessioni infinite senza
    un server). Il ricollegamento avviene in una piccola FINESTRA POPUP:
    fai login lì e l'app si aggiorna da sola, senza ricaricare. Se il
    dispositivo blocca i popup, l'app passa automaticamente al metodo a
    pagina intera (redirect).
  - CONFRONTO AUTOMATICO: all'avvio e al collegamento, l'app confronta
    i dati locali con il backup su Drive. Se sul cloud c'è una versione
    PIÙ RECENTE (es. modificata su un altro dispositivo), te lo dice e
    ti chiede se vuoi caricarla ("Carica dal cloud"). Niente viene
    sovrascritto senza la tua conferma. Se invece è più aggiornato il
    dispositivo, l'app riallinea il cloud da sola (se l'auto-backup è
    attivo). Questo evita di perdere dati per sbaglio.
  - In OPZIONI, sotto "Backup cloud", la voce "Sincronizzazione" mostra
    se il backup è "Allineato" o se ci sono modifiche ancora da salvare.
  - Resta valido il principio "vince l'ultimo che salva": se modifichi
    in contemporanea su due dispositivi senza sincronizzare, l'ultimo
    backup sovrascrive il precedente. Il confronto automatico serve
    proprio a segnalartelo prima.
  - Il file "casa-app-backup.json" è visibile nel Drive dell'utente: lo
    si recupera dall'app con "Ripristina", ma volendo si può anche
    scaricare a mano da Google Drive.

----------------------------------------
 7. PRIMO AVVIO
----------------------------------------

Al primo avvio l'app è vuota e mostra la schermata "CONFIGURA CASA":
  - Crea almeno un piano/area (es. "Piano Terra").
  - Crea almeno una stanza (es. "Cucina").
  - Il pulsante "Entra nell'app" si attiva solo quando esiste almeno
    un piano/area E almeno una stanza.

Dopo il primo avvio, piani/aree e stanze si gestiscono da OPZIONI
(le stanze stanno dentro ciascun piano: Gestione piani/aree -> tocca un piano).
Dopo un Reset dati l'app torna alla schermata "CONFIGURA CASA".

----------------------------------------
 8. DIAGNOSTICA (in OPZIONI)
----------------------------------------

In OPZIONI trovi un box "Diagnostica" che mostra lo stato dei dati:
versione app, numero piani/aree, stanze, lavori, idee, voci spesa,
chiave di salvataggio e data dell'ultimo aggiornamento. Utile per
controllare al volo cosa c'è salvato sul dispositivo.
